# 安全与回滚

高风险功能会要求输入 `YES`。不确定时先创建备份，再执行修改。

## 备份范围

备份会尽量覆盖 SSH、主机名、Nginx/Caddy、443 单入口、DNS、证书、Cloudflare Token、Docker、Fail2ban、sysctl 和 3x-ui 关键配置。

备份里可能包含私钥、面板数据库和 API Token，不要公开分享。

## 常用入口

```text
主菜单 [16 配置备份与回滚]
```

常用操作：

- 创建全量配置备份。
- 查看现有备份列表。
- 从备份一键回滚。
- 查看或编辑脚本已应用配置。

## 高风险操作

这些操作前建议先确认云安全组、保留当前 SSH 会话，并准备好快照或救援控制台：

- 修改 SSH 端口。
- 开启仅密钥登录。
- 修改防火墙规则。
- 切换 443 入口模式。
- 修改证书和反向代理配置。
- 启用流量达量关机。

复杂故障处理见 [失联与回滚急救手册](recovery-runbook.md)。
